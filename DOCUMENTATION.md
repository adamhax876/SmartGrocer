# 📘 SmartGrocer SaaS - الدليل الشامل (Documentation)

مرحباً بك في الدليل الشامل لنظام **SmartGrocer**. يهدف هذا الملف لشرح كل صغيرة وكبيرة في النظام، هيكل الملفات، وظائفها، وكيفية التعديل عليها بسهولة لضمان استمرارية عملك واستطاعتك تطوير النظام في المستقبل.

---

## 🏗️ 1. هيكل المشروع (Project Structure)
النظام مبني باستخدام **Node.js** و **Express** للواجهة الخلفية (Backend)، و **HTML/CSS/JS** نقي للواجهة الأمامية (Frontend)، مع قاعدة بيانات **MongoDB**.

المسار الرئيسي للمشروع هو:
`c:\Users\VIP\.gemini\antigravity\scratch\smartgrocer\`

### 📂 المجلدات الرئيسية:
- **`public/`**: يحتوي على كل ما يراه المستخدم النهائي (HTML, CSS, JS, Images).
- **`routes/`**: يحتوي على ملفات الواجهة الخلفية (الـ API) التي تعالج الطلبات.
- **`models/`**: يحتوي على تصاميم الجداول في قاعدة البيانات (مثل جدول المستخدمين، المنتجات).
- **`middleware/`**: يحتوي على طبقات الحماية والتأكد من الصلاحيات (مثل التأكد من أن المستخدم مسجل الدخول، أو أنه أدمن).
- **`utils/`**: يحتوي على أدوات مساعدة כמו إرسال الإيميلات (`email.js`)، والرسائل اللحظية (`sse.js`)، وجلب الإعدادات.

## 🗂️ 2. شرح ملفات الواجهة الأمامية (`public/`)
كل ما يخص تصميم الموقع وكيف يتفاعل معه المستخدم موجود هنا.

### 📄 ملفات الـ HTML الأساسية:
| الملف | الوظيفة | كيفية التعديل عليه |
|-------|---------|-------------------|
| `index.html` | **الصفحة الرئيسية:** صفحة الهبوط التي تعرض مميزات النظام وتجذب العملاء. | لتغيير النصوص التسويقية، ابحث عن `translations` في أسفل الملف وعدل النصوص العربية والإنجليزية. |
| `login.html` | **تسجيل الدخول:** تتيح للتجار الدخول للوحة التحكم. | مرتبطة برابط الـ API (`/api/auth/login`). |
| `signup.html` | **حساب جديد:** تسجيل المتاجر الجديدة. ترسل إيميل تفعيل. | |
| `dashboard.html` | **لوحة تحكم التاجر:** تعرض الإحصائيات (الأرباح، المبيعات) عبر رسوم بيانية. | للتعديل على التصميم أو القائمة الجانبية، ابحث عن وسم `<aside>`. |
| `pos.html` | **نقطة البيع الكاشير (POS):** شاشة الكاشير السريعة. | لإضافة أزرار أو تعديل الفاتورة، ابحث عن `receiptModal` أو دالة `renderCart()`. تستخدم العملة بشكل ديناميكي من الإعدادات. |
| `products.html` | **إدارة المنتجات:** لإضافة وتعديل وحذف المنتجات ورفع الصور. | ترسل البيانات لـ `/api/products`. |
| `inventory.html` | **المخزون (Inventory):** لمراقبة الكميات وقيمة المخزن بشكل كامل. | |
| `sales.html` | **المبيعات:** تعرض سجل جميع الفواتير السابقة والحالية. | |
| `admin/dashboard.html` | **لوحة الإدارة (السوبر أدمن):** للمدير العام فقط لمراقبة كل المتاجر. | |
| `admin/settings.html` | **إعدادات النظام العامة:** لتغيير العملة ($، جنيه، الخ) واسم الموقع وإيقاف النظام (وضع الصيانة). | |

### 📁 مجلدات الواجهة الأمامية الفرعية:
- **`css/global.css`**: ملف الألوان الأساسية والتنسيقات. إذا أردت تغيير ألوان الموقع (الوضع الليلي والنهاري) عدّل قيم الـ `--primary`, `--bg`, `--text` هنا في أعلى الملف.
- **`js/app.js`**: **العقل المدبر للواجهة الأمامية**.
  - يحتوي على دالة `api()` التي تتحدث مع الباك إند.
  - يحتوي على دالة `getToken()` للتحقق من تسجيل الدخول.
  - يحتوي على نظام **الإشعارات اللحظية (SSE)** في دالة `initRealtimeNotifications()` لتلقي وعرض الشاشات المنبثقة من الإدمن.
  - يحتوي على `formatMoney()` لتوحيد عرض العملة في كل الموقع بناءً على إعدادات الأدمن.
- **`images/`**: يحتوي على الصور الثابتة، وأهمها `logo.png`.
  - **لتغيير اللوجو:** ببساطة قم بمسح الصورة القديمة وصع الصورة الجديدة تحت اسم `logo.png` في نفس هذا المجلد.

## ⚙️ 3. شرح ملفات الواجهة الخلفية (Backend)
الواجهة الخلفية تتعامل مع قاعدة البيانات وتتأكد من الحماية.

### 📄 الملف الرئيسي (`server.js`)
- **وظيفته:** هو قلب النظام الذي يربط كل شيء. يفتح المنفذ (Port 3000)، يتصل بقاعدة MongoDB، ويحدد أين تذهب كل الروابط (Routes).
- **التعديل عليه:** إذا أردت إضافة قسم جديد بالكامل (مثل "الموردين")، ستقوم بإضافة مساره هنا مثل: `app.use('/api/suppliers', ...)`.

### 📂 مجلد `routes/` (مسارات الـ API):
| الملف | الوظيفة | ملحوظات هامة |
|-------|---------|--------------|
| `auth.js` | تسجيل الدخول، إنشاء الحساب، إعادة تعيين كلمة المرور، توليد التوكن (JWT). | مسؤول عن إرسال إيميل التفعيل الـ OTP. |
| `products.js`| CRUD المنتجات (إضافة، تعديل، حذف، بحث منتجات متجر معين). | |
| `sales.js` | إنشاء الفواتير (من الـ POS) وخصم الكمية المتوفرة من المخزون تلقائياً عند الدفع. | إذا أردت إضافة طريقة دفع (فيزا)، تعدلها هنا. |
| `admin.js` | صلاحيات المدير، جلب جميع المتاجر، تعديل إعدادات النظام، إرسال رسائل للنظام (Popups). | تذكر: يرسل الرسالة ويستدعي `sendRealtimeNotification`. |
| `messages.js`| يجلب صندوق الوارد والإشعارات للتاجر عند فتح الموقع. | |

### 📂 مجلد `models/` (قاعدة البيانات):
تمثل تصاميم (Schemas) الجداول في قاعدة بيانات MongoDB:
- `User.js`: بيانات المتجر، الإيميل، الباسورد (مشفر بـ bcrypt)، العملة، التوكن والـ OTP.
- `Product.js`: اسم المنتج، الباركود، السعر، الكمية، صورة المنتج. مرتبط بالـ User (صاحب المتجر).
- `Sale.js`: تفاصيل الفاتورة، المنتجات المباعة، الإجمالي، طرق الدفع.
- `Setting.js`: الإعدادات العامة للموقع (العملة، وضع الصيانة).
- `Message.js`: الإشعارات والرسائل.

## 🚀 4. طريقة أخذ نسخة احتياطية (Backup)
لضمان عدم ضياع تعبك والموقع الخاص بك، قمنا ببرمجة نظام للنسخ الاحتياطي السريع.

**خطوات النسخ الاحتياطي (تعمل على الويندوز):**
1. تم دمج سكريبت في ملف يسمى `backup.js` في مجلد المشروع الرئيسي.
2. لأخذ نسخة، ببساطة افتح محرر الأوامر (Terminal أو CMD) وتأكد أنك بداخل مجلد المشروع.
3. اكتب الأمر التالي:
   `node backup.js`
4. سيقوم النظام بضغط كامل ملفات المشروع (أكواد وصور) وتجاهل ملفات `node_modules` الثقيلة، وسيقوم بوضع ملف الـ ZIP مباشرة على **سطح المكتب (Desktop)** الخاص بك مع تاريخ وساعة النسخة!
