# الدليل الشامل لرفع مشروع SmartGrocer على GitHub والاستضافة (VPS / RDP)

هذا الدليل يوضح لك الخطوات المتبقية لنشر مشروعك للعامة بشكل احترافي وآمن.

## أين نحن الآن؟ (مراجعة الأمان)
لقد قمت بإجراء فحص أمني شامل للمشروع بأكمله:
1. **تم غلق ثغرات تجاوز الصلاحيات:** جميع الـ APIs الخاصة بالإدارة العليا لا يمكن الدخول عليها نهائياً من قبل موظفي "الدعم الفني". حتى لو تمكن الموظف من كتابة الرابط يدوياً، سيطرده السيرفر (Backend).
2. **إخفاء واجهة المستخدم (UI):** تم إخفاء جميع الأزرار والتبويبات الجانبية (Sidebar) من حسابات الدعم الفني بناءً على طلبك، لكي لا يعرفوا حتى ما هي خصائص النظام الداخلي.
3. **حماية قواعد البيانات:** تم إصلاح جميع أخطاء (NaN) وهجمات الـ Null في الفواتير وحفظ البيانات لضمان عدم تعطل النظام.
4. **العزل في حالة انتهاء الاشتراك:** النظام أصبح يطبق وضع (Read-Only) بصرامة، ولن يستطيع أي عميل التلاعب بالتواريخ لإضافة منتجات مجانية.

---

## 🚀 الخطوة الأولى: رفع المشروع على GitHub

الآن، يجب أن تحفظ نسخة آمنة من مشروعك على GitHub للرجوع إليها مستقبلاً.

1. افتح الـ Terminal (سطر الأوامر) داخل مجلد المشروع (VS Code).
2. تأكد من وجود ملف يسمى `.gitignore` وأنه يحتوي على السطر `node_modules/` و `.env` لكي لا يتم رفع كلمات المرور الخاصة بك للعامة.
3. قم بكتابة الأوامر التالية بالترتيب:

```bash
git add .
git commit -m "الإصدار النهائي لنظام الـ SaaS واشتراكات الدفع والدعم الفني"
git push origin main
```
*ملاحظة: إذا كنت تستخدم واجهة GitHub Desktop، يمكنك ببساطة الضغط على زر (Commit to main) ثم (Push origin).*

## 🌍 الخطوة الثانية: رفع النظام للعامة على سيرفر (RDP أو Linux VPS)

بما أنك أشرت لامتلاكك RDP، فإليك كيف تجعل الموقع متاحاً لجميع الناس على الإنترنت بدلاً من `localhost`.

### الخيار الأول: تشغيل النظام داخل الـ RDP (Windows Server) الخاص بك
إذا كان الـ RDP يعمل بنظام تشغيل ويندوز، اتبع الآتي:

1. **نقل الملفات:** قم بعمل نسخ لمجلد المشروع (أو سحبه من GitHub) بداخل الـ RDP.
2. **تثبيت المتطلبات:** قم بتحميل برنامج **Node.js** وتثبيته بداخل الـ RDP تماماً مثل جهازك الشخصي.
3. **التشغيل الدائم:** لا تعتمد على `npm start` العادية لأنها ستغلق بمجرد قفل الشاشة، قم بفتح شاشة الـ Terminal أو الـ PowerShell كمسؤول (Run as administrator) داخل الـ RDP واكتب:
   ```bash
   npm install -g pm2
   pm2 start server.js --name "smartgrocer"
   ```
   *(أداة PM2 ستضمن أن موقعك سيظل يعمل 24 ساعة حتى لو قمت بإغلاق نافذة الـ RDP، وستعيد تشغيله تلقائياً في حال إعادة تشغيل السيرفر).*
4. **ضبط مسار الإنترنت (Port Forwarding):** 
   يجب عليك إعداد جدار الحماية (Windows Firewall) في الـ RDP لفتح الـ Port رقم `3000` (Inbound Rules) للعامة.
   الآن، سيتمكن أي شخص في العالم من الدخول للموقع عبر كتابة:
   `http://[IP-Address-Of-RDP]:3000`
5. **ربط الدومين (اختياري):** يمكنك شراء دومين (Domain) مثل `smartgrocer.com` وربطه بالـ IP الخاص بالـ RDP من خلال إعدادات DNS، ثم استخدام برنامج مثل `Nginx` لتوجيه الـ Port 3000 إلى پورت 80.

### الخيار الثاني: نقل المشروع لاستضافة احترافية (Hostinger / DigitalOcean / Vercel)
هذا هو الخيار الأفضل تجارياً. في حال حجزت استضافة (Linux VPS)، يمكنك استخدام منصات أو أدوات سهلة مثل **Nginx Proxy Manager** لتشغيل موقعك. 
يظل استخدام قاعدة بيانات `MongoDB Atlas` (التي نستخدمها بالفعل في مشروعك) كافياً جداً في جميع الحالات.
